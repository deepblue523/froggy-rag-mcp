Chunking Settings

Chunk Size

characters

Typical: 256–1024. Smaller = more precise but more queries; larger = more context but fuzzier matches.

Chunk Overlap

characters

Typical: 32–128. Helps avoid splitting important sentences/sections across chunks.

Smart chunking

(1.0 = only if shorter than chunk size)

Hierarchical mode stores extra “coarse” chunks for the same spans; use retrieval dedupe so search returns the single best match per group.

Chunking LLM (optional)

Ollama (e.g. http://127.0.0.1:11434) or OpenAI-compatible base URL (e.g. https://api.openai.com/v1). Set the model name below (Ollama tag or OpenAI model id). Used only during ingest when enabled.

Full width for long names (registry paths, dated OpenAI snapshots, vision tags).

Minimum Chunk Size

characters
tokens

Skip tiny/low-signal chunks. Chunks below these thresholds will be filtered out.

Max Chunks Per Document

(0 = unlimited)

Helps avoid one huge doc dominating the index. Set to 0 for unlimited chunks per document.

Embedding Model

Select the embedding model to use. Examples: "text-embedding-3-large", "nomic-embed-text", "gte-large".

Normalize Embeddings

Whether to L2-normalize vectors. Usually true for cosine similarity.

These settings will be applied to newly ingested documents.

Retrieval Settings

Top K

neighbors

Typical: 5–30. How many neighbors to return from the search.

Score Threshold

(0 = no threshold)

Drop results below a minimum similarity. Useful to avoid hallucinating when nothing is relevant.

Max Returned Chunks Per Document

(0 = unlimited)

Avoid multiple very-similar chunks from the same doc. E.g. max_chunks_per_doc = 3.

Group by Document / Return Full Documents

Whether to group results by document and/or return full documents with scores.

Max Context Tokens

(0 = unlimited)

How many tokens you're willing to feed into the LLM; you can use this to cap top_k dynamically.

Hierarchical chunk dedupe

When hierarchical chunking is on, fine and coarse chunks share a group id; this returns only the highest-scoring hit per group.

These settings will be applied to search queries.

General Settings

Window Behavior

When enabled, minimizing or closing the window will hide it to the system tray instead of closing the application. You can restore the window by clicking the tray icon.

App updates

Check GitHub Releases for a newer version. Download progress and release notes also appear on the Updates tab.

Server Settings

Server Port

The port number on which the MCP server will listen for connections. Default is 3000.

Auto-Start Server

When enabled, the MCP server will automatically start when the application launches using the configured port.

These settings will be applied when starting the server.

Metadata & Filtering Settings

Time Range / Since

days (0 = no filter)

Prioritize documents created or updated within the specified number of days. Set to 0 to disable time-based filtering.

Time Decay

When enabled, multiply similarity scores by a time-based decay factor to prioritize fresher documents.

Time Decay Half Life

days

The number of days after which a document's similarity score is reduced by half. Documents older than this will have progressively lower scores.

These settings will be applied to search queries.

App updates

Update checks run only in the installed release build (not in development mode).

The app checks GitHub Releases for a newer version on startup and every four hours. When an update is found, it downloads in the background and can install when you restart (or on quit if you dismiss the prompt).

Publish installers with npm run build:publish (requires GH_TOKEN or GITHUB_TOKEN for uploads).

Namespaces

Each namespace uses its own folder under your app data data/ directory (vector store, watched files, etc.). Names may contain letters, numbers, - and _.
