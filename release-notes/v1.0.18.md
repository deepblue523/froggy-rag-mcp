# Froggy RAG MCP v1.0.18 — Google Web Search MCP tool, web-augmented LLM Passthrough, and Markdown-aware chunking

**Release tag:** `v1.0.18`  
**Package version:** 1.0.18  
**Type:** Feature release

---

## Summary

This release adds a **Google Custom Search** integration that exposes a new **`web_search`** MCP tool and, when enabled, **mixes web result snippets into the retrieved context** for **LLM Passthrough** alongside the local vector-store hits. A dedicated **Web Search** settings page collects the API key, search engine ID, results-per-search, and request timeout, and is persisted through its own IPC path so concurrent saves cannot clobber freshly typed credentials. Chunking gains a **Markdown-aware** path: Markdown documents are now subdivided on **block boundaries** (paragraphs, setext headings, fenced code blocks, tables) and reassembled with **block-level overlap** instead of mid-paragraph slicing, producing cleaner Markdown chunks and fewer split tables / code fences.

## Highlights

### Web search

- **New MCP tool `web_search`** — backed by **Google Custom Search**. Input: `query` (string, required) and optional `numResults` (1–10). Returns the query, a `results` array (`title`, `url`, `displayLink`, `snippet`, `formattedUrl`), a pre-formatted `context` block, and Google's `searchInformation`. Errors bubble up as JSON-RPC `Invalid params` (e.g. when the API key or engine ID is not configured).
- **Web-augmented LLM Passthrough** — when **`llmPassthroughIncludeWebResults`** is on, both the direct passthrough path and the inbound HTTP chat proxy fetch web results for the resolved RAG query and combine them with vector-store hits under labeled sections (**`### Local vector store results`** / **`### Web search results`**). System preamble and empty-context fallback now reference both retrieval sources. Web search failures are added as a warning (`Web search skipped: …`) and never abort the chat.
- **`src/main/services/web-search.js`** — new module with `searchGoogleCustomSearch`, `formatWebResultsForContext`, `getGoogleCustomSearchConfig`, `assertGoogleCustomSearchConfigured`. Number of results clamps to **1–10**, timeout clamps to **1–60 s**, and requests use `AbortController` for cancellation.

### Settings → Web Search

- **New settings page** under **Settings → Web Search** with fields for **API key** (password input, blank means "keep existing"), **Search engine ID (cx)**, **Results per search** (default 5), **Timeout (seconds)** (default 15), and the **Include web results in LLM Passthrough context** toggle.
- **Isolated save path** — new IPC channel **`save-web-search-settings`** (`electronAPI.saveWebSearchSettings`) and `RAGService.saveWebSearchSettings(patch)` only mutate web-search keys and persist them, so unrelated background saves (splitter drag, LLM tab autosave, etc.) cannot overwrite credentials being typed in this panel. Blank API key submissions preserve the existing stored key.
- **New defaults** in `getDefaultSettings()`: `googleCustomSearchApiKey: ''`, `googleCustomSearchEngineId: ''`, `googleCustomSearchNumResults: 5`, `googleCustomSearchTimeoutSeconds: 15`, `llmPassthroughIncludeWebResults: false`.

### Markdown-aware chunking

- **`subdivideMarkdownUnit`** (new, in `document-chunk-strategies.js`) — when a Markdown unit exceeds the target chunk size, it is split on **Markdown block boundaries** instead of generic sentence boundaries. **Fenced code blocks** and **GFM tables** are treated as **indivisible** unless a single block alone exceeds `maxChars` (in which case the generic `subdivideUnit` is used as a fallback). **Setext headings** (`===` / `---` underlines) are kept attached to their title line.
- **`_mergeMarkdownStringsWithOverlap`** (new, in `document-processor.js`) — Markdown chunks are reassembled by **packing whole blocks** until `chunkSize` is reached and then taking **trailing whole blocks** for overlap (rather than slicing characters mid-paragraph). Blocks are joined with `\n\n` to keep Markdown structure intact, and chunks are tagged with `docProfile: 'markdown'` metadata.
- **Routing** — `DocumentProcessor.chunkText` now picks the Markdown subdivider/merger automatically when `profile.kind === 'markdown'`; non-Markdown documents continue to use the existing `subdivideUnit` / `_mergeStringsWithOverlap` path.

### UI / misc

- **Sidebar** — the top-level **LLM** node is renamed to **Passthrough** to match the actual feature surface.
- **README (dev mode)** — `npm run dev` no longer implies auto-opening DevTools; pass **`--open-devtools`** to launch Electron with the inspector open.

## Upgrade notes

- **No migration required** — new web-search settings default to empty / disabled, and existing installs keep behaving as before until you fill in credentials and toggle **Include web results in LLM Passthrough context**.
- **To enable web augmentation:**
  1. Create a Google **Programmable Search Engine** and a Custom Search **API key**.
  2. Open **Settings → Web Search**, paste the **API key** and **Search engine ID (cx)**, choose **Results per search** (1–10) and **Timeout (seconds)** (1–60), and click **Save Web Search settings**.
  3. To have inbound LLM Passthrough mix web snippets into the model context, also tick **Include web results in LLM Passthrough context** before saving.
- **MCP clients** can now call **`web_search`** directly (returns snippets only — no fetched page bodies). The tool requires Web Search to be configured in the desktop app; otherwise the call returns a parameter error explaining what is missing.
- **Re-ingestion (optional)** — Markdown documents that previously chunked across paragraph boundaries will only pick up the new block-aware boundaries on re-ingest. Existing indexes continue to work unchanged; remove and re-add Markdown sources if you want them re-chunked under the new strategy.
