## Summary

This release adds **optional HTTP request logging** for traffic hitting the **MCP REST** server and **inbound LLM passthrough** listeners. Each completed request can be recorded with method, path, status code, duration, and source; entries persist next to **`settings.json`** as **`~/froggy-rag-mcp/mcp-request-log.json`**, respect a configurable **retention window** (days), and cap total stored entries. **Settings → Logging** exposes enable/disable and retention. The **MCP Server** area in the UI uses a **tabbed layout** so you can switch between the existing URL/endpoint summary and a **Request log** view with refresh and live updates via IPC. Several long-form **`docs/`** guides that were redundant or out of date with the current tree were removed to reduce noise; operational detail lives in the **README** and in-repo code/tests.

## Highlights

### MCP and passthrough HTTP request logging

- **`src/main/services/mcp-request-log.js`** — **`appendRequestLog`**, **`getRequestLogs`**, and **`attachHttpRequestLogger`** (Express middleware on **`res.finish`**) record **`method`**, **`path`**, **`statusCode`**, **`durationMs`**, and **`source`** (`mcp-rest` vs inbound listener labels). Logging runs only when **`mcpRequestLoggingEnabled`** is **`true`** in app settings.
- **Retention** — **`mcpRequestLogRetentionDays`** (default **7**, clamped **1–3650**) prunes entries by timestamp on read and append. Up to **10,000** entries are kept on disk; older overflow is dropped from the tail of the array after pruning.
- **Integration** — **`mcp-service.js`** and **`passthrough-inbound-server.js`** attach the middleware; **`main.js`** forwards new entries so the UI can update in real time.

### Settings and persistence

- **`settings-files.js`** defaults: **`mcpRequestLoggingEnabled: false`**, **`mcpRequestLogRetentionDays: 7`**. New **Settings → Logging** page in **`index.html`** / **`app.js`** for the checkbox and retention number input; values persist with the rest of app settings when the settings modal is saved.

### Renderer, IPC, and preload

- **`ipc-handlers.js`** — **`get-mcp-server-request-logs`** returns recent entries (limit clamped); **`mcp-service`** **`request-log`** events are bridged to the renderer as **`mcp-server-request-log`**.
- **`preload.js`** — **`getMCPServerRequestLogs(limit)`** and subscription helper for **`mcp-server-request-log`**.
- **MCP Server panel** — Tab **Endpoints** vs **Request log**; hint copy explains that logs include REST and inbound passthrough when logging is enabled.

## Upgrade notes

- **From v1.2.0:** No migration required. New settings keys default to logging **off** and **7**-day retention; existing **`settings.json`** merges with defaults.
- **Privacy / disk:** Enabling logging writes request metadata (paths may include query strings) to **`~/froggy-rag-mcp/mcp-request-log.json`**. Turn logging off or shorten retention on shared machines or when paths are sensitive.
- **Prior release notes** that linked removed **`docs/`** files: use **README** or **`release-notes/v1.2.0.md`** for historical context; those standalone guides are no longer in the tree.
- **Ship checklist:** Run **`npm run version:minor`** (or edit **`package.json`** and tag consistently), **`npm test`**, then build/publish per your usual **`electron-builder`** flow.
