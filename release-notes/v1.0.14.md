# Froggy RAG MCP v1.0.14 — Markdown ingestion and chunking

**Release tag:** `v1.0.14`  
**Package version:** 1.0.14  
**Type:** Feature and ingestion fix

---

## Summary

This release improves how **Markdown** (`.md`, `.markdown`, `.mdx`) is discovered during directory ingest and how it is **chunked** for the vector store, using the same intelligent pipeline as other structured document types.

## Highlights

- **Structure-aware Markdown chunking** — logical splits respect fenced code blocks (so `#` inside a shell snippet does not start a new section), ATX headings (`#`–`######`), setext headings, YAML front matter as its own leading unit, and thematic breaks (`---`, `***`, etc.) without confusing setext underlines with horizontal rules. Short heading-only fragments are merged with the following section when that helps retrieval. Very long, heading-sparse documents get a paragraph-style fallback before sentence-based subdivision.
- **Default chunk size hint for Markdown** — profile suggests a slightly larger character window (3200) so typical “section + prose” units stay coherent through overlap merge.
- **Directory scanning and watches** — recursive and non-recursive directory ingest, directory file listings, and chokidar **add / change / unlink** handlers now include `.md`, `.markdown`, and `.mdx` via a single shared extension list aligned with the document processor.
- **File picker** — the Files screen “add file” dialog accepts the same Markdown extensions.

## Upgrade notes

- **Re-ingest** Markdown-backed corpora if you want embeddings to reflect the new chunk boundaries; existing chunks are unchanged until you remove and re-add files or re-run directory ingest.
